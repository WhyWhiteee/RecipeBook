## Техническое задание (ТЗ)
### Проект: Система управления рецептами
### Стек: Django 5 + Django REST Framework (DRF) + Vue.js 3 + SQLite

---

## 1. Общие сведения

- **Название системы**: Система управления рецептами (Recipe Management System)
- **Назначение**: создание, хранение и поиск рецептов, управление избранным, классификация по кухням.
- **Тип приложения**: веб-приложение (Frontend SPA + Backend REST API).
- **Технологический стек**:
  - Backend: **Django 5**, **DRF**
  - Frontend: **Vue.js 3**, **Pinia**, **Vue Router**, **Axios**
  - База данных: **SQLite** (`db.sqlite3`)
- **Целевая аудитория**: домашние кулинары, авторы рецептов; администратор для модерации.

---

## 2. Предметная область

Система предназначена для пользователей, которые хотят:
- создавать рецепты (ингредиенты, шаги, фото, кухня);
- искать рецепты по ингредиентам и кухне;
- сохранять понравившиеся рецепты в избранное;
- просматривать подробную информацию о рецепте и управлять своими рецептами.

---

## 3. Роли и права пользователей

- **Пользователь (user)**:
  - регистрируется и авторизуется;
  - просматривает рецепты, ищет, фильтрует;
  - создаёт/редактирует/удаляет **только свои** рецепты;
  - ведёт список избранных рецептов;
  - управляет своим профилем.

- **Администратор (admin)**:
  - управляет пользователями (при необходимости блокирует);
  - управляет справочником кухонь;
  - модерирует рецепты (скрывает/удаляет нарушающие правила).

---

## 4. Функциональные требования (15+)

Формулировки в виде «Система должна...».

1. Система должна позволять пользователю регистрироваться.
2. Система должна позволять пользователю авторизоваться и выходить из аккаунта.
3. Система должна предоставлять пользователю просмотр списка публичных рецептов.
4. Система должна предоставлять CRUD-операции для рецептов (создание, просмотр, редактирование, удаление) в рамках прав доступа.
5. Система должна позволять добавлять к рецепту ингредиенты с количеством, единицей измерения и примечанием.
6. Система должна позволять добавлять к рецепту шаги приготовления с порядком выполнения.
7. Система должна позволять прикреплять фотографии к рецепту и/или к шагам (опционально).
8. Система должна позволять назначать рецепту категорию кухни.
9. Система должна поддерживать поиск рецептов по названию/описанию.
10. Система должна поддерживать поиск/фильтрацию рецептов по одному или нескольким ингредиентам.
11. Система должна поддерживать фильтрацию рецептов по кухне.
12. Система должна поддерживать сортировку списка рецептов (по дате создания, названию, времени приготовления).
13. Система должна позволять пользователю добавлять рецепты в избранное.
14. Система должна позволять пользователю просматривать и редактировать список избранных (добавлять/удалять).
15. Система должна предоставлять подробную страницу рецепта (ингредиенты, шаги, фото, автор, кухня).
16. Система должна ограничивать редактирование/удаление рецепта только автором или администратором.
17. Система должна предоставлять администратору управление кухнями (создание/редактирование/удаление).
18. Система должна предоставлять администратору возможность модерации рецептов (скрытие/удаление).

---

## 5. Нефункциональные требования (7+)

1. Интерфейс должен быть адаптивным (responsive) для мобильных и десктопных устройств.
2. Система должна корректно обрабатывать ошибки и информировать пользователя (валидация, сообщения).
3. Система должна обеспечивать безопасность учетных данных: хранить пароль только в виде хэша.
4. Система должна обеспечивать разграничение доступа по ролям и авторству данных.
5. Время отклика API для основных операций (список/поиск/детали) не должно превышать **500 мс** при типовой нагрузке (например, до 1000 рецептов).
6. Система должна обеспечивать целостность данных (нельзя создать рецепт без названия, без шагов и без ингредиентов).
7. Система должна быть разворачиваемой локально без интернета после установки зависимостей.
8. Система должна вести серверное логирование технических ошибок (без паролей и чувствительных данных).

## 6. Архитектура системы

### 6.1 Компоненты

- **Frontend (Vue.js 3)**:
  - Vue Components (Views/Components): UI страниц и элементов.
  - Vue Router: маршрутизация.
  - Pinia Store: состояние и клиентская логика.
  - Axios: HTTP клиент.

- **Backend (Django + DRF)**:
  - Django URLs / DRF Routers: маршрутизация API.
  - Views / ViewSets: обработка запросов, права, фильтры.
  - Serializers: валидация и преобразование данных ↔ JSON.
  - Models: ORM-модели.
  - Middleware: CORS, auth и пр.

- **Database**:
  - SQLite (`db.sqlite3`).

### 6.2 Потоки данных

- Frontend → Backend: **HTTP request** (JSON body/query params)
- Backend → Frontend: **HTTP response** (JSON)
- Backend (Models) → SQLite: **ORM queries** (SELECT/INSERT/UPDATE/DELETE)

## 7. Модель данных

### 7.1 Сущности

- **User (Пользователь)**
- **Cuisine (Кухня/категория кухни)**
- **Recipe (Рецепт)**
- **Ingredient (Ингредиент)**
- **RecipeIngredient (Ингредиент в рецепте)** — промежуточная таблица M:N
- **RecipeStep (Шаг приготовления)**
- **Favorite (Избранное)** — промежуточная таблица M:N

### 7.2 Связи

- User 1:N Recipe (авторство)
- Cuisine 1:N Recipe
- Recipe 1:N RecipeStep
- Recipe M:N Ingredient через RecipeIngredient
- User M:N Recipe через Favorite

## 8. REST API (Django REST Framework)

- **Базовый префикс**: `/api/`
- **Формат**: JSON
- **Авторизация**: токен/JWT (в заголовке `Authorization: Bearer <token>`)

### 8.1 Таблица endpoints

| Раздел | Метод | URL | Назначение | Тело запроса (пример) | Ответ (пример) |
|---|---|---|---|---|---|
| Auth | POST | `/api/auth/register/` | Регистрация | `{email,password,password_confirm,display_name}` | `{id,email,display_name}` |
| Auth | POST | `/api/auth/login/` | Логин, получение токенов | `{email,password}` | `{access,refresh}` |
| Auth | GET | `/api/auth/me/` | Профиль текущего пользователя | — | `{id,email,display_name,role}` |
| Recipes | GET | `/api/recipes/` | Список рецептов + фильтры/поиск | — (query params) | `{count,next,previous,results:[...]}` |
| Recipes | POST | `/api/recipes/` | Создание рецепта | `{title,description,cuisine_id,...,ingredients:[...],steps:[...]}` | `Recipe (detail)` |
| Recipes | GET | `/api/recipes/{id}/` | Детали рецепта | — | `Recipe (detail)` |
| Recipes | PATCH | `/api/recipes/{id}/` | Обновление рецепта | `{title,is_public,...}` | `Recipe (detail)` |
| Recipes | DELETE | `/api/recipes/{id}/` | Удаление рецепта | — | `204 No Content` |
| Ingredients | GET | `/api/ingredients/` | Список/поиск ингредиентов | — (`?search=...`) | `[{id,name},...]` |
| Cuisines | GET | `/api/cuisines/` | Список кухонь | — | `[{id,name,description?},...]` |
| Favorites | GET | `/api/favorites/` | Список избранного | — | `[{recipe...},...]` |
| Favorites | POST | `/api/favorites/` | Добавить в избранное | `{recipe_id}` | `{user_id,recipe_id,created_at}` |
| Favorites | DELETE | `/api/favorites/by-recipe/{recipe_id}/` | Удалить из избранного | — | `204 No Content` |
| Photos | POST | `/api/photos/` | Загрузка фото | `multipart/form-data(file,caption?)` | `{id,url,caption,uploaded_by}` |

> Примечание: фактические эндпоинты могут быть реализованы через `ViewSet` + `Router` (DRF), а часть — как `APIView`.

## 9. Интерфейс (UI)

### 9.1 Обязательные экраны

1. **Вход/Регистрация**
   - поля email/пароль, регистрационные поля, кнопки входа/регистрации.
2. **Главная**
   - быстрый поиск, подборки, кухни, карточки рецептов.
3. **Список рецептов**
   - поиск, фильтры (ингредиенты/кухня), сортировка, карточки, пагинация.
4. **Детальный просмотр рецепта**
   - фото, метаданные, ингредиенты, шаги, кнопка избранного, (для автора) редактирование/удаление.
5. **Форма рецепта (создание/редактирование)**
   - поля рецепта, ингредиенты (таблица/строки), шаги, загрузка фото, сохранение.
6. **Профиль**
   - данные пользователя, выход, вкладки “мои рецепты”, “избранное”.
7. **Избранное**
   - карточки рецептов, удаление из избранного.

## 10. Глоссарий

- **Рецепт (Recipe)** — карточка блюда с описанием, ингредиентами, шагами, фото и категорией кухни.
- **Ингредиент (Ingredient)** — элемент справочника продуктов (например, “молоко”, “яйцо”).
- **Ингредиент в рецепте (RecipeIngredient)** — связь рецепт–ингредиент с количеством и единицей измерения.
- **Шаг приготовления (RecipeStep)** — упорядоченная инструкция по приготовлению.
- **Кухня (Cuisine)** — категория рецепта (итальянская, японская и т.д.).
- **Избранное (Favorite)** — список рецептов, сохраненных пользователем.
- **DRF (Django REST Framework)** — библиотека для построения REST API на Django.
- **SPA** — одностраничное приложение (Vue.js), работающее через API.
- **ORM** — слой доступа к данным Django, формирующий запросы к базе данных.
